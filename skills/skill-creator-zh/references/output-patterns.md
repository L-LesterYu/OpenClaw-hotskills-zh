# 输出模式设计指南

本文档描述了创建一致、高质量输出的最佳实践。

## 模板模式

### 基本模板结构

```markdown
## 输出模板

### 标题
[输出标题]

### 摘要
[简要摘要]

### 详情
[详细内容]

### 后续步骤
[推荐的操作]
```

### 变量替换

```markdown
## 报告模板

**项目名称**：{project_name}
**日期**：{date}
**状态**：{status}

### 进度
{progress_details}
```

## 质量标准

### 输出检查清单

- [ ] 格式正确
- [ ] 内容完整
- [ ] 无拼写错误
- [ ] 结构清晰
- [ ] 引用准确

### 质量级别

1. **草稿** - 初始版本，可能需要修订
2. **审核** - 等待用户反馈
3. **最终** - 已批准，可交付

## 示例模式

### 报告格式

```markdown
# [主题] 分析报告

## 执行摘要
[2-3 句概述]

## 关键发现
1. [发现 1]
2. [发现 2]
3. [发现 3]

## 详细分析
[深入分析]

## 建议
- [建议 1]
- [建议 2]

## 附录
[支持材料]
```

### 清单格式

```markdown
# [任务名称] 清单

## 准备
- [ ] 项目 1
- [ ] 项目 2

## 执行
- [ ] 步骤 1
- [ ] 步骤 2

## 验证
- [ ] 检查 1
- [ ] 检查 2
```

## 最佳实践

1. **一致性** - 对相同类型的内容使用相同的格式
2. **可读性** - 使用清晰的标题和结构
3. **完整性** - 包含所有必要信息
4. **可操作性** - 提供明确的后续步骤
